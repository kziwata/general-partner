# Modes — Detailed Workflows

## Think（壁打ち）

### When

ユーザーが判断に迷っている、方針を決めたい、複数案を比較したい。

### Steps

1. **確認** — 目的・制約・期限で不明点があれば1〜2問（なければスキップ）
2. **認識合わせ** — 現状理解を3〜5文で要約
3. **選択肢** — 2〜4案。各案: メリット / デメリット / 前提
4. **推奨** — 推奨案と理由（押し付けない）
5. **次のアクション** — 1〜3個に絞る
6. **記録** — ユーザーが希望すれば `sessions/` に保存

### Avoid

- いきなり実装に入らない
- 選択肢を1つに絞りすぎない（最低2案）

---

## Organize（情報整理）

### When

散らばったメモ、会話ログ、調査結果を構造化したい。

### Steps

1. **入力の種類**を判定: 生メモ / 会話 / 検索結果 / 混在
2. **4分類**:
   - 決定事項 → `DECISIONS.md`
   - タスク → `TASKS.md`
   - 参考情報 → `NOTES.md`
   - 未分類 → `INBOX.md`（整理後に空に近づける）
3. **重複・矛盾**を指摘
4. `README.md` の索引を必要に応じて更新

### INBOX 処理ルール

- INBOX は受信箱。整理セッションで必ず振り分ける
- 1トピック = 1セクション（長い場合は `sessions/` へ）

---

## Search（調査）

### When

外部知識、最新情報、技術比較が必要。

### Steps

1. 調査目的と成功条件を明確化
2. ソース選択:
   - Web → 一般・最新情報
   - コードベース → プロジェクト内
   - `partner/` / `docs/partner/` → 過去メモ
3. 収集・要約（出典を必ず付ける）
4. 統合レポート（templates.md の Search テンプレート）
5. 不確実な点を明示
6. 必要なら `NOTES.md` に保存

### Avoid

- 出典なしの断言
- 古い情報を最新として提示（日付を確認）

---

## Task（タスク管理）

### When

やることの整理、優先順位、進捗確認。

### Steps

1. タスクを抽出（会話 + 既存 `TASKS.md`）
2. 優先度・期限・ブロッカーを付与
3. 会話中は **TodoWrite** で追跡
4. セッション跨ぎは **TASKS.md** に反映
5. 完了時は Done セクションへ移動（2週間後にアーカイブ可）

詳細: [task-management.md](task-management.md)

---

## Mode transitions

| From | To | Trigger |
|------|-----|---------|
| Think | Task | 「じゃあこれをやる」と決まった |
| Think | Search | 判断に必要な情報が不足 |
| Search | Organize | 調査結果をファイルに残す |
| Organize | Task | 整理中にタスクが出た |
| Any | Implement | ユーザーが実装を明示 — 整理を保存してから委譲 |
